# Network & Web Security Classification Pipeline 🛡️

An end-to-end, modular Machine Learning Operations (MLOps) pipeline designed to detect and classify cyber threats across different layers of the OSI model. 

This project goes beyond simple model training by implementing a fully fault-tolerant, object-oriented architecture. It features automated data ingestion, strict schema validation, dynamic feature engineering, and automated experiment tracking, proving readiness for production-level deployment.

---

## 🚀 Release Versions

### [v1.0.0] - Web Security & Phishing Detection Engine (Stable)
**Focus:** Application-Layer Threat Detection (Layer 7)

A highly robust, static-analysis security engine designed to identify malicious webpages and phishing attempts. It evaluates complex URL structures, lexical features, and third-party domain metadata to neutralize threats before a user interacts with them.
* **Architecture:** Batch-processing ML pipeline utilizing a custom dataset of URL features.
* **Key Achievements:**
    * Fully modular ETL pipeline connecting directly to MongoDB.
    * Automated model artifact generation (`.pkl` files) using Scikit-Learn.
    * Complete experiment tracking and hyperparameter tuning logged via MLflow.

### [v2.0.0] - Live Network Intrusion Detection System (In Active Development)
**Focus:** Real-Time Network-Layer Threat Detection (Layer 3/4)

This version upgrades the core pipeline to handle live, high-velocity infrastructure defense. Utilizing the industry-standard CIC-IDS2017 dataset format, this engine captures and classifies raw network traffic flows to detect DDoS, Brute Force, and Botnet attacks as they happen.
* **Architecture:** Streaming-data ML pipeline analyzing continuous network packet flows.
* **Key Upgrades:**
    * **Live Packet Sniffing:** Integration with `cicflowmeter` to intercept live traffic.
    * **Real-Time Feature Extraction:** On-the-fly calculation of 78 complex flow statistics (e.g., Flow IAT, Packet Length Mean).
    * **Advanced Data Transformation:** Dynamic handling and neutralization of `NaN` and `Infinity` anomalies generated by microsecond flow errors.

## 🏗️ Modular ML Pipeline Architecture

This project is built using object-oriented programming principles, separated into four distinct, highly decoupled stages. The entire pipeline is wrapped in a custom logging and exception handling framework to ensure easy debugging and continuous monitoring.

**Foundation: Custom Logging & Exception Handling**
Instead of standard print statements, the pipeline uses a centralized, custom Python logger to track the execution flow across all modules. Custom exception classes are implemented to catch and isolate pipeline failures gracefully, preventing silent errors and making debugging straightforward.

**1. Data Ingestion**
Bypasses local CSV files by connecting directly to **MongoDB**. This module programmatically fetches the raw dataset from the cloud, batches it, and splits it into training and testing sets for downstream use.

**2. Data Validation**
Before any processing happens, the ingested data passes through a validation check. This stage verifies that the incoming data matches the expected schema (checking data types and ensuring no missing columns), preventing the pipeline from crashing due to unexpected data changes.

**3. Data Transformation**
A dedicated ETL module that cleans and prepares the data. It mathematically handles edge cases (like neutralizing `NaN` and `Infinity` values), applies statistical scaling, and serializes these preprocessing steps into a `.pkl` artifact. This ensures the exact same transformations are applied during real-time inference.

**4. Model Training & Tracking**
Trains the classical machine learning models while integrating with **MLflow**. Instead of manual evaluation, this stage automatically tracks hyperparameters, logs performance metrics (Accuracy, Precision, F1-Score), and saves the best-performing model as a ready-to-use artifact.

## ⚙️ Installation & Setup

**1. Clone the repository**
```bash
git clone [https://github.com/yourusername/network-security-pipeline.git](https://github.com/yourusername/network-security-pipeline.git)
cd network-security-pipeline

**2. Create a virtual environment**
```bash
python -m venv venv
source venv/bin/activate  # On Windows use `venv\Scripts\activate`
```

**3. Install dependencies**
```bash
pip install -r requirements.txt
```

**4. Set up environment variables**
Create a `.env` file in the root directory and add your MongoDB connection string and MLflow tracking URI:
```env
MONGO_DB_URL="your_mongodb_cluster_url"
MLFLOW_TRACKING_URI="your_mlflow_uri"
```

**5. Run the Pipeline**
Execute the main training pipeline to ingest data, validate, transform, and train the model:
```bash
python main.py
```
